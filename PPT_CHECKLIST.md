## 🎨 Design Standards

**Colors (Use consistently):**
- [ ] Primary: #1a1a1a (charcoal) - Headers
- [ ] Secondary: #ff6b35 (orange) - Highlights
- [ ] Accent: #4ecdc4 (teal) - Important info
- [ ] Background: #ffffff (white) or #f5f5f5 (light gray)
- [ ] Text: #333333 (dark gray)

**Fonts:**
- [ ] Headers: Montserrat Bold (or similar)
- [ ] Body: Open Sans Regular (or Arial)
- [ ] Code: Courier New (monospace)
- [ ] Size: Headers 40pt+, Body 18pt+

**Layout:**
- [ ] 16:9 aspect ratio (modern standard)
- [ ] Consistent margins (1 inch all sides)
- [ ] Left-aligned text with visual hierarchy
- [ ] One diagram/image per slide (not too crowded)

## ⚠️ Common Mistakes to Avoid

- [ ] **Don't** read text directly from slides (boring!)
- [ ] **Don't** use small fonts (students can't read from back)
- [ ] **Don't** crowd too much info on one slide
- [ ] **Don't** forget to practice with actual PowerPoint
- [ ] **Don't** leave demo untested (Murphy's law!)
- [ ] **Don't** oversee technical jargon (explain briefly)
- [ ] **Don't** ignore Q&A preparation
- [ ] **Don't** rush through slides (2-3 min per slide is good)
- [ ] **Don't** forget to backup presentation (USB + Cloud)
- [ ] **Don't** show code on slides (too small, distracting)

## 🚀 On Presentation Day

### 30 Minutes Before
- [ ] Arrive early, test projector
- [ ] Check laptop connection to projector
- [ ] Open PowerPoint and test advance buttons
- [ ] Have presentation on desktop (not in folder)
- [ ] Close unnecessary browser tabs
- [ ] Put phone on silent

### 15 Minutes Before
- [ ] Do deep breathing (calm nerves)
- [ ] Review first 3 slides
- [ ] Check microphone/speaker levels
- [ ] Have water bottle ready
- [ ] Smile and have confidence!

### During Presentation
- [ ] Make eye contact with judges/class
- [ ] Speak slowly and clearly
- [ ] Use hand gestures
- [ ] Pause for emphasis
- [ ] Watch time (should fit in allocated time)
- [ ] Show enthusiasm about your project

### After Presentation
- [ ] Thank the audience
- [ ] Open for questions
- [ ] Answer confidently (say "I don't know" if needed, not guessing)
- [ ] Don't over-explain
- [ ] End with main takeaway
